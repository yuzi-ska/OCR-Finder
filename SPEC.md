# OCR Finder - 规格说明

## 1. 项目概览

**项目名称：** OCR Finder
**项目类型：** Python / Tkinter 桌面应用
**核心能力：** 递归扫描本地目录，从图片中提取文本，支持多关键词匹配（AND/OR/NOT），并将匹配文件链接到输出目录。

## 2. 技术栈

- **语言：** Python 3.10+
- **GUI：** Tkinter
- **OCR 引擎：** RapidOCR (ONNX Runtime) - CPU only
- **进程管理：** psutil
- **打包：** Nuitka Windows standalone package

## 3. CLI 规格

```text
usage: ocr_finder.py [-h] -t TARGET -s SOURCE [-o OUTPUT] [-m MODE] 
                      [--output-mode OUTPUT_MODE] [-c CPU] [-v]
```

### 参数说明

- `-t`, `--target`
  - 目标关键词（多个用逗号分隔），必填
- `-s`, `--source`
  - 源目录，必填
- `-o`, `--output`
  - 输出目录，默认 `./output`
- `-m`, `--mode`
  - 匹配模式：`and`（全部匹配）、`or`（任一匹配）、`not`（都不匹配）
  - 默认 `or`
- `--output-mode`
  - 输出方式：`hardlink`（硬链接）、`symlink`（软链接）、`copy`（复制）
  - 默认 `hardlink`
- `-c`, `--cpu`
  - CPU 限制百分比 (1-100)
  - 默认 `50`
- `-v`, `--verbose`
  - 输出详细日志（包括每个文件的 OCR 结果）
- `-h`, `--help`
  - 显示帮助

## 4. 运行时配置

### 4.1 支持的输入类型

仅支持图片文件：
- `.jpg`, `.jpeg`, `.png`, `.bmp`, `.gif`, `.tiff`, `.tif`, `.webp`

不支持 PDF 文件。

### 4.2 多关键词匹配

关键词分隔符：
- 英文逗号 `,`
- 英文分号 `;`
- 中文逗号 `，`
- 中文分号 `；`
- 竖线 `|`

匹配模式：
- `or` - 任一关键词匹配即可（默认）
- `and` - 所有关键词都必须匹配
- `not` - 所有关键词都不匹配

匹配规则：
- 大小写不敏感
- 忽略首尾空白
- 合并连续空白

### 4.3 进程优先级

通过调整进程优先级控制 CPU 占用：

| 设置 | CPU 限制 | 优先级 |
|------|----------|--------|
| 低 | 30% | 最低优先级 |
| 中 | 50% | 低于正常优先级 |
| 高 | 90% | 正常优先级 |

### 4.4 输出方式

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| 硬链接 | 创建指向相同数据的新文件 | 同分区，节省空间 |
| 软链接 | 创建符号链接 | 跨分区 |
| 复制 | 完整复制文件 | 需要独立副本 |

### 4.5 离线约束

- 程序完全离线运行
- RapidOCR 模型已内置，无需下载
- 打包 EXE 可直接运行

## 5. 功能规格

### 5.1 文件扫描

1. 递归遍历源目录
2. 当输出目录位于源目录内部时，自动跳过输出目录
3. 仅纳入图片文件
4. 生成输入文件列表供后续处理

### 5.2 文本提取

- 图片输入先做基本可读性校验
- 交给 RapidOCR 执行识别
- 提取结果做归一化处理

### 5.3 匹配判定

1. 对目标文本与提取文本做归一化
2. 根据匹配模式判断是否命中
3. 命中时提取匹配的关键词

### 5.4 输出复制

1. 自动创建输出目录（如不存在）
2. 根据输出方式处理文件（硬链接/软链接/复制）
3. 跨分区时自动回退到普通复制
4. 使用文件内容哈希避免重复复制
5. 若文件名冲突但内容不同，自动追加 `_1`、`_2` 等后缀

## 6. 打包规格

### 6.1 构建方式

- 入口脚本：`build_cpu.bat`
- 基于 Nuitka 生成 Windows standalone `.dist` 目录
- 输出到 `release/`
- 可执行文件名：`OCRFinder.exe`

### 6.2 打包内容

- `OCRFinder.exe`
- Python 运行时
- RapidOCR + ONNX Runtime
- psutil
- Tkinter/Tcl

### 6.3 体积

- 实际：~240MB

3. ✅ 程序仅支持图片 OCR（不支持 PDF）
4. ✅ 程序支持多关键词匹配（AND/OR/NOT）
5. ✅ 程序支持进程优先级选择

8. ✅ 部分文件失败时不会中断整个扫描

## 9. GUI 规格

### 界面元素

1. **界面语言选择** - 中文/English 下拉框
2. **目标文本输入框** - 支持多关键词
3. **源文件夹选择** - 文本框 + 浏览按钮
4. **输出文件夹选择** - 文本框 + 浏览按钮
5. **匹配模式选择** - 单选按钮（OR/AND/NOT）
6. **进程优先级下拉框** - 低/中/高
7. **输出方式下拉框** - 硬链接/软链接/复制
8. **详细日志复选框**
9. **开始/停止按钮**
10. **进度条**
11. **状态标签**
12. **日志输出区**
13. **打开输出目录按钮**
14. **清空日志按钮**

### 多语言支持

- 中文（默认）
- 英文（可切换）
